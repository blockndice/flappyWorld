Les scripts sont chargés dans l'ordre : sound.js → shop.js → game.js. Chaque fichier expose ses fonctions au scope global ; game.js les consomme.

Rôle de chaque fichier JS

FichierLignesResponsabilité
sound.js172Déclaration des 16 sons, moteur de chargement avec timeout secours 5 s, volume maître, déverrouillage AudioContext mobile, chaînage musiques (startRun → travelMusic → dead → resumeMusic).
shop.js534Catalogue des 17 articles (SHOP_ITEMS), état joueur actif (skin, trail, trick, jump, sndJump), systèmes de particules trail/jump/trick, animations skins spéciaux (Phosphor, Multicolor), logique achat/équipement/preview, rendu canvas du shop.
game.js1 719Constantes physiques, machine d'états, boucle loop(), update(), render(), sprites pixel-art, UI pop-up (score / leaderboard / try-again), responsive scaling, favicon dynamique, persistance localStorage.

Ordre d'exécution au démarrage

1. sound.js  — crée les Audio(), lance le chargement, installe _unlockAudio
2. shop.js   — définit SHOP_ITEMS, relit localStorage (équipements, pièces)
3. game.js   — relit localStorage (scores, version), setFavicon(), init(), loop()
             → le jeu entre immédiatement dans l'état 'intro1'
             → soundsReady reste false tant que les sons ne sont pas prêts
             → le premier clic/touch déverrouille l'audio (politique navigateur)

Machine d'états du jeu

Diagramme de flux

╔══════════════════════════════════════════════════════════════╗ ║ CYCLE DE JEU ║ ╚══════════════════════════════════════════════════════════════╝ [ intro1 ] ──── page 2 → clic "Free Run" ────► [ intro2 ] ▲ │ │ t ≥ 280 frames │ ▼ │ [ countdown ] │ │ │ 240 frames (~4 s) │ ▼ │ [ playing ] │ │ │ collision / sol │ ▼ │ [ dead ] │ │ │ bird sort de l'écran (y > H+80) │ ▼ │ [ score ] │ page 1 → 2 → 3 │ │ │ No (→ init()) ◄──────────────────────┘ Yes → countdown

Les 6 états en détail

intro1

Écran titre animé. L'oiseau flap au centre, des oiseaux de fond traversent. 4 sous-pages : accueil · menu · historique · shop.

intro2

Cinématique scriptée (~280 frames). La formation d'oiseaux traverse l'écran. Aucun input possible. Se termine automatiquement.

countdown

Décompte 5→4→3→2→1→Go! (6 × 40 frames = 240). L'oiseau se positionne. Le sol défile déjà.

playing

Partie active. Gravité, tuyaux, collecte de pièces, tricks, trail. Collision → die().

dead

Frames 1–20 : shake d'impact. Frame 21+ : bird projeté hors écran avec physique + particules d'impact × 5.

score

Pop-up 3 pages : GAME OVER + score · TOP 10 · Try Again? (Yes/No). Clic avance les pages.

Boucle principale

function loop() {
  coinTick++;              // anime la pièce (rotation cosinus)
  // défilement des nuages (hors dead/score)
  update();               // physique + logique selon l'état courant
  render();               // effacement + dessin de tous les sprites
  requestAnimationFrame(loop);
}

Ordre de rendu dans render()

  1. Shake d'écran si dead frame 1–10 (translate aléatoire)
  2. sprBg() — ciel dégradé + soleil + nuages
  3. Tuyaux + pièces (sprPipe, sprCoin)
  4. sprGround() — bande herbe + terre scrollante
  5. Oiseaux de fond (renderBgBirds) — uniquement en intro1
  6. Particules trail + jump + trick (depuis shop.js)
  7. Oiseau joueur (sprBird) avec translate + scale
  8. drawUI() — HUD, pop-ups, menus selon l'état

Responsive scaling

function updateScale() {
  const mobile = window.matchMedia('(pointer: coarse)').matches;
  if (mobile) {
    // Mobile : remplit toute la hauteur — la largeur déborde et est clippée
    const scale = window.innerHeight / H; // H = 600
    canvas.style.transform       = `scale(${scale})`;
    canvas.style.transformOrigin = 'center center';
    // canvas reste 400×600 en layout → pas de déformation flex
  } else {
    // Desktop : letterbox classique
    const scale = Math.min(window.innerWidth / W, window.innerHeight / H);
    canvas.style.width  = `${W * scale}px`;
    canvas.style.height = `${H * scale}px`;
  }
}
AppareilStratégieRésultat
DesktopMath.min(vw/W, vh/H)Letterbox, aspect ratio préservé
Mobile portraitvh/H + transform scalePlein écran en hauteur, bords latéraux clippés
Mobile paysagevh/H (plus petit ici)Remplit la hauteur, marges noires latérales
Tablette portraitvh/HRemplit la hauteur (largeur canvas ≤ vp width)

Coords écran → canvas (canvasPos)

function canvasPos(clientX, clientY) {
  const r = canvas.getBoundingClientRect(); // rect visuel après CSS transform
  return [(clientX - r.left) * W / r.width,
           (clientY - r.top)  * H / r.height];
}

Plein écran

  • Desktop/tablette : bouton #fsBtn → requestFullscreen() / exitFullscreen() avec préfixe webkit
  • iOS : ne supporte pas l'API Fullscreen — couvert par les meta PWA (apple-mobile-web-app-capable) pour le mode "ajout à l'écran d'accueil"
  • Le bouton est masqué sur les petits téléphones (pointer: coarse + max-width: 500px)
